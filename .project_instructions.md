# Form Claw - Project Instructions

## Overview
Automated PDF Form-Filling System monitoring dashboard. Monitors russakbot@gmail.com, processes forms, fills them with family data, and replies.

## Daemon Architecture (Webhook-only, no polling)
- **Gmail Watch Renewal** (task fa40ec28c): Runs every 2 days, renews Pub/Sub watch
- **Form Processor** (task f9a21765a): Event-driven webhook trigger, processes forms when Pub/Sub pushes
  - Webhook URL: `https://apps.abacus.ai/api/webhooks?abacus_deployment_token=MTMxODA3ODA6MTM5NjI1OTA6NTMyMTA6ZA.bd451fc3e446486225da48634634cd0c&v=1`
- **Health Check** (task aa3562d2d): Runs every 3 hours, calls /api/health/check which sends email alerts if daemons are stale
- **REMOVED**: Gmail Poller (14e8862670) and main polling daemon (de057bf2f) - disabled June 2026

## Health Monitoring
- `/api/health/heartbeat` - POST: daemon heartbeat reporting (watch_renew, form_process), GET: public health status
- `/api/health/check` - GET: checks Watch Renewal staleness and sends email alerts
- HEARTBEAT_TOKEN used for auth between daemons and dashboard
- Staleness threshold: Watch Renewal 3d
