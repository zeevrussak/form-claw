# Form Claw - Project Instructions

## Overview
Automated PDF Form-Filling System dashboard. Receives emails at `formclaw@savlil.com` via Cloudflare Email Workers, processes PDF form attachments, fills them with Russak family data, and replies via Resend API.

## Architecture (Cloudflare Email + Resend — NO Gmail API)
- **Incoming**: Cloudflare Email Routing → Email Worker → POST webhook → Form Processor Daemon
- **Outgoing**: Resend API (`formclaw@savlil.com`)
- **NO Gmail API, NO Pub/Sub, NO OAuth tokens needed**

## Cloudflare Email Worker
- Source: `/home/ubuntu/shared/cloudflare-email-worker/`
- User deploys to Cloudflare via `npx wrangler deploy`
- Receives at `formclaw@savlil.com`, parses MIME, base64-encodes PDFs, POSTs to webhook
- Whitelist check built in (also checked by daemon)

## Health Monitoring
- `/api/health/heartbeat` - POST: daemon heartbeat (form_process only), GET: public health status
- `/api/health/check` - GET: checks form processor staleness, sends alert emails with dedup
- HEARTBEAT_TOKEN for auth between daemons and dashboard
